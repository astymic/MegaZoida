# Deployment and Testing Guide

## Окружение для разработки
1. **Node.js**: Требуется версия 18.x или выше.
2. **Технологический стек**: Vite, TypeScript, HTML5 Canvas API (или Phaser / React по необходимости), CSS3. Без тяжелых движков, максимальная оптимизация под Web/Mobile.
3. **Сборка**: Проект собирается в статические файлы. Ванильный JS/TS позволяет добиться 60 FPS на любых мобильных устройствах.

## Как запустить локально
1. Клонируйте репозиторий: `git clone https://github.com/astymic/MegaZoida.git`
2. Перейдите в папку: `cd MegaZoida`
3. Установите зависимости: `npm install`
4. Запустите dev_server: `npm run dev`
5. Откройте в браузере (обычно `http://localhost:5173`).
6. **Для тестирования мобильной версии**: Откройте DevTools в браузере (F12) и включите режим Device Toolbar (Mobile View).

## Как тестировать
1. **Ручное тестирование (Геймплей)**:
   - Проверьте управление (WASD, клики по экрану/виртуальный джойстик).
   - Убедитесь, что автоатака корректно выбирает цель.
   - Соберите опыт, получите Level Up — проверьте появление окна выбора из 3 баффов.
   - Проверьте логику замены оружия, если у персонажа уже 6 слотов.
2. **Производительность**:
   - Следите за FPS. Если на экране 200 мобов и FPS падает ниже 30, нужна оптимизация (объединение отрисовки, spatial hash grid для коллизий).

## Деплой (Deployment)
1. Проект может быть размещен на любой платформе статического хостинга (GitHub Pages, Vercel, Netlify).
2. Выполните сборку: `npm run build`.
3. Загрузите содержимое папки `dist` на хостинг.
4. **Mobile (PWA/Cordova/Capacitor)**: Для превращения в нативное приложение для Android/iOS можно использовать Capacitor. Достаточно проинициализировать его поверх папки `dist`.
