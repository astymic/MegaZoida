# Руководство: IDE для мобильной веб-разработки игр

Так как `MegaZoida` создается исключительно для мобильных телефонов, работа инженера на ПК должна эмулировать мобильную среду. 

## Рекомендуемая IDE — Visual Studio Code (VS Code)
Это стандарт индустрии для написания веб-приложений и игр на TypeScript/Vite. 

### Необходимые плагины (Extensions) в VS Code:
1. **ESLint** — проверка чистоты TypeScript-кода и выявления ошибок.
2. **Prettier** — автоматическое выравнивание и форматирование кода.
3. **Three.js Snippets** — горячие клавиши для быстрого написания 3D сцен (понадобятся при переходе на 3D!).

## Локальное тестирование на ПК
Чтобы игра ощущалась как на телефоне при разработке:
1. Запустите в терминале VS Code: `npm run dev`
2. Откройте Google Chrome по адресу `http://localhost:5173`.
3. Запустите **Chrome DevTools** (клавиша `F12`).
4. Включите **Device Mode** (комбинация `Ctrl + Shift + M`).
5. Выберите модель телефона из выпадающего списка (например, iPhone 12 Pro или Galaxy S20).
6. Обновите страницу (`F5`), и браузер начнет имитировать Touch-события (тапы вместо кликов) и мобильные пропорции экрана (Viewport).

## Локальное тестирование прямо на ВАШЕМ телефоне
Если вы хотите поиграть и проверить управление непосредственно со своего смартфона:
1. Измените скрипт запуска в файле `package.json`:
   `"dev": "vite --host"`
2. Запустите `npm run dev`. В терминале выведется IP-адрес вашей Wi-Fi сети (что-то вроде `http://192.168.1.101:5173`).
3. Убедитесь, что ПК и телефон подключены к **одной и той же Wi-Fi сети**.
4. Откройте на телефоне любой мобильный браузер (Safari или Chrome) и вбейте этот IP-адрес. Игра откроется прямо на смартфоне в реальных условиях платформы!

## Финальная мобильная сборка в Native Приложение (.apk / .ipa)
Хоть это и веб-игра, мы превратим её в нативное мобильное приложение с помощью библиотеки **Capacitor** (от компании Ionic):
1. Выполните сборку: `npm run build` (создастся папка `dist` с игрой).
2. Заверните папку `dist` через Capacitor: выполните `npx cap add android` и `npx cap open android`. 
3. Вы получите нативный проект Android Studio, готовый к финальной компиляции в `.apk` под публикацию в Google Play!
